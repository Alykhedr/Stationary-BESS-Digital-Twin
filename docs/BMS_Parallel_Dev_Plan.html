BMS Parallel Development Plan

Stationary BESS Digital Twin — Sony US26650FTC1 (LFP)

Project: Stationary-BESS-Digital-Twin Reference: Schimpe (2018) TUM Dissertation Date: June 2026 Authors: Ali & Diaa

Core Principle

Agree on interfaces first — build in parallel — integrate at gates.
Every module is a black box defined only by its function signature. Once signatures are signed off, both people work independently. Stubs (functions that return correct-shaped neutral outputs) let the full model run end-to-end from day one.

The plan is split into sprints (parallel, independent work) and gates (together, integration + pass/fail check). A new sprint never starts on a failing gate.

Step 0 — Interface Session Together · ~3 hours

Do this before splitting. Nothing else starts until this is committed and both people confirm main.m runs clean.

0.1 — Function Signatures (the contracts)

These are frozen after this session. A signature change is a discussion, not a unilateral commit.

% ── SENSOR MODEL ────────────────────────────────────────────────────────
% IN:  true_val     – scalar or vector (V, A, or °C)
%      cfg_sensor   – struct: .bias, .noise_std, .q_lsb, .tau_h
%      dt_h         – timestep [h]
% OUT: meas_val     – same size as true_val (measured value with error)
function meas_val = bms_sensor_model(true_val, cfg_sensor, dt_h)

% ── OCV INVERSE ─────────────────────────────────────────────────────────
% IN:  V_ocv        – measured/inferred OCV [V]
%      I_last_sign  – last current direction: +1 (charge) / -1 (discharge)
%      SOC_hint     – current SOC estimate (initial guess for interpolation)
%      cfg          – bms_config struct
% OUT: SOC_ocv      – SOC read from OCV curve [0..1]
%      in_steep     – true if dOCV/dSOC large enough for correction (bool)
function [SOC_ocv, in_steep] = bms_ocv_inverse(V_ocv, I_last_sign, SOC_hint, cfg)

% ── SOC ESTIMATOR ───────────────────────────────────────────────────────
% IN:  I_meas       – measured current [A]  (+ = charge)
%      V_meas       – measured terminal voltage [V]
%      T_meas       – measured temperature [°C]
%      dt_h         – timestep [h]
%      cfg          – bms_config struct
%      state_in     – struct carrying memory: .SOC_est, .Q_est, .t_rest_h,
%                     .I_last_sign, .anchor_soc, .anchor_Ah
% OUT: SOC_est      – estimated SOC [0..1]
%      state_out    – updated state struct (same fields as state_in)
function [SOC_est, state_out] = bms_soc_estimator(I_meas, V_meas, T_meas, dt_h, cfg, state_in)

% ── SOH ESTIMATOR ───────────────────────────────────────────────────────
% IN:  I_meas       – measured current [A]
%      SOC_est      – current SOC estimate [0..1]
%      V_meas       – measured voltage [V]
%      dt_h         – timestep [h]
%      cfg          – bms_config struct
%      state_in     – struct: .Q_est, .anchor1_soc, .anchor1_Ah, .counting
% OUT: Q_est        – estimated capacity [Ah]
%      SOH_est      – Q_est / Q_nom [0..1]
%      state_out    – updated state struct
function [Q_est, SOH_est, state_out] = bms_soh_estimator(I_meas, SOC_est, V_meas, dt_h, cfg, state_in)

% ── DYNAMIC LIMITS (SOP) ────────────────────────────────────────────────
% IN:  SOC_est      – estimated SOC [0..1]
%      Q_est        – estimated capacity [Ah]
%      V_meas       – measured voltage [V]
%      T_meas       – measured temperature [°C]
%      R_est        – estimated internal resistance [Ω]
%      dt_h         – timestep [h]
%      cfg          – bms_config struct
% OUT: I_ch_max     – max allowable charge current [A]  (≥ 0)
%      I_dis_max    – max allowable discharge current [A]  (≥ 0)
function [I_ch_max, I_dis_max] = bms_limits(SOC_est, Q_est, V_meas, T_meas, R_est, dt_h, cfg)

% ── MODE STATE MACHINE ──────────────────────────────────────────────────
% IN:  I_meas       – measured current [A]
%      V_meas       – measured voltage [V]
%      T_meas       – measured temperature [°C]
%      SOC_est      – estimated SOC [0..1]
%      I_ch_max     – limit from bms_limits [A]
%      I_dis_max    – limit from bms_limits [A]
%      cfg          – bms_config struct
%      state_in     – struct: .mode (string), .t_idle_h, .fault_flags
% OUT: mode         – string: 'INIT'/'STANDBY'/'CHARGE'/'DISCHARGE'/'DERATE'/'FAULT'
%      I_ch_out     – I_ch_max clamped to 0 in FAULT [A]
%      I_dis_out    – I_dis_max clamped to 0 in FAULT [A]
%      state_out    – updated state struct
function [mode, I_ch_out, I_dis_out, state_out] = bms_mode(I_meas, V_meas, T_meas, ...
                                                    SOC_est, I_ch_max, I_dis_max, cfg, state_in)

% ── BMS INIT ────────────────────────────────────────────────────────────
% IN:  cfg          – bms_config struct
%      V_meas_t0    – measured voltage at t=0 (used to seed SOC_est via OCV)
%      seed         – integer for rng (reproducibility)
% OUT: biases       – struct: .b_V, .b_I, .b_T  (frozen offsets drawn once)
%      est_state    – initial estimator state struct (for soc + soh estimators)
%      mode_state   – initial mode state struct
function [biases, est_state, mode_state] = bms_init(cfg, V_meas_t0, seed)

0.2 — Stub Implementations

Each function above gets a stub in BMS/ with the exact signature returning neutral values:

FunctionStub output
bms_sensor_modelreturns true_val unchanged (zero error)
bms_ocv_inversereturns SOC_hint, in_steep = false
bms_soc_estimatorreturns state_in.SOC_est, state_out = state_in
bms_soh_estimatorreturns cfg.cell.Q_nom, SOH_est = 1.0
bms_limitsreturns cfg.cell.I_chg_max_cont, cfg.cell.I_dis_max_cont

bms_initreturns zero biases, default state structs

0.3 — bms_config.m

Written once, committed, read-only for both people after this session. Contains all thresholds (voltages, temperatures, currents, SOC windows, sensor error parameters, balancing parameters). Neither person hard-codes a threshold inside their own function — they read from cfg.

🔒 Gate 0 Exit Criterion
  • main.m runs end-to-end with stub BMS — no errors
  • Outputs are bit-identical to pre-BMS baseline
  • Both people pull, run, and confirm independently
  • Commit tagged: gate-0-stubs

Sprint 1 — Parallel Build ~3–4 days

No shared files edited. Track A owns estimation files. Track B owns protection files and the .slx model.

Track A — Estimation Pipeline

  • bms_ocv_inverse.m — build first (needed by SOC estimator)
  • bms_sensor_model.m — bias + noise + quantization
  • bms_soc_estimator.m — CC + OCV-snap corrector
  • bms_soh_estimator.m — anchor-pair capacity bookkeeping
  • Test/test_soc_estimator.m — open-loop test script

No Simulink touched. Pure MATLAB functions + test scripts only.

Track B — Protection & Dispatch Pipeline

  • bms_init.m — build first (seeds state structs)
  • bms_limits.m — dynamic SOP with derating curves
  • bms_mode.m — mode state machine (MATLAB function)
  • Battery_sim_2.slx — add BMS subsystem, rewire dispatch
  • Slim current_limiter.m — remove SOC window clamp (moves to bms_limits)
  • Test/test_limits.m, Test/test_state_machine.m

Owns the .slx model during Sprint 1. Track A does not touch Battery_sim_2.slx.

Gate 1 — Connect Estimation to Protection Together · ~2 hours

Trigger: Both tracks have passed their own self-tests and pushed to main.

What happens: Replace Track A stubs in the Simulink BMS subsystem with real implementations. Wire MeasBus output into the estimators. Wire SOC_est and Q_est into limits and mode. Run the 10-year simulation.

Sprint 2 — Analysis & Pack Preparation After Gate 1

Track A — Estimation Analysis

  • Bias injection experiment: sweep b_I ∈ {1, 5, 10, 20} mA → plot SOC_est worst-case error & anchor-recovery behaviour
  • Quantify: how often do valid SOH anchor pairs occur in the 10-year profile?
  • Optional: EKF as second estimator for thesis comparison

Track B — Pack Preparation

  • Vectorize plant cell model: scalar → 16×1 blocks per module (16s12p)
  • Add per-block parameter scatter (capacity ±0.5%, R ±2%, self-discharge ±5%)
  • bms_balancing.m: dissipative balancing logic (33 Ω bleed, 5 mV threshold)
  • Wire balance_cmd back into per-block SOC derivatives

Gate 2 — Pack Integration Together · ~2 hours

Connect the vectorized pack plant to the existing BMS. Only bus dimensions change — no algorithm equations change. Verify block voltages diverge under scatter then re-converge under balancing. E_bal accumulator is nonzero and plausible. With scatter disabled, balancing never triggers.

Rules — How We Avoid Blocking Each Other

  1. File ownership during a sprint is exclusive. Track A owns BMS/bms_*estimator.m, bms_sensor_model.m, bms_ocv_inverse.m. Track B owns bms_limits.m, bms_mode.m, bms_init.m, and Battery_sim_2.slx. bms_config.m is read-only for both after Step 0.
  2. Stubs stay until your real implementation passes its own self-test. Never push a half-working function. If it breaks the other person's model run, that's a blocker for them.
  3. Gate sessions are time-boxed at 2 hours. If the gate isn't passing at the end, log what broke and schedule a follow-up. Don't let an unresolved integration issue drift.
  4. Function signatures are frozen after Step 0. If you discover a signature needs to change, flag it as a discussion item — don't change it unilaterally and don't start working around it silently.
  5. Tests travel with the function. When you push a real implementation, the corresponding test script ships in the same commit. A function without a passing test is treated as a stub.
